# 性能优化

OpenClaw 的性能特性和优化策略。

## 📊 性能指标

| 指标 | 数值 |
|------|------|
| Gateway 内存占用 | < 200MB |
| 消息入站延迟 | < 10ms |
| Lane 排队延迟 | < 1ms |
| 上下文构建 | < 50ms |
| LLM 调用延迟 | 2-10s (取决于 Provider) |

## 🚀 优化策略

### 1. 上下文压缩

```
对话历史 < 80% Token 预算 → 不压缩
对话历史 > 80% Token 预算 → Compaction 压缩旧消息
```

### 2. 流式响应

```
非流式：等待完整响应（5-10s）→ 一次性发送
流式：   每个 token 到达即推送（< 100ms 首字延迟）
```

### 3. 工具执行

```
工具超时：15 秒默认
长命令：设置 timeout=300
超时后 SIGTERM → 5s → SIGKILL
```

### 4. 缓存

```
- 模型列表缓存：减少 Provider API 调用
- 工具结果缓存：重复工具调用可复用
- 通道连接池：复用 WebSocket 连接
```
